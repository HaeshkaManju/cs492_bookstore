# API Reference

**Related Stories:** All backend stories  
**Related Tasks:** S2-7 (Inventory API), all endpoint implementations

---

## 1. Overview

The API follows RESTful conventions with:
- JSON request/response bodies
- JWT Bearer token authentication
- Consistent error response format
- OpenAPI 3.0 documentation (auto-generated by FastAPI)

**Base URL:** `https://api.bookstore.example.com/api/v1`

---

## 2. Authentication

### 2.1 Header Format
```
Authorization: Bearer <access_token>
```

### 2.2 Token Endpoints

| Endpoint | Method | Auth | Description |
|----------|--------|------|-------------|
| `/auth/register` | POST | None | Create customer account |
| `/auth/login` | POST | None | Authenticate and get token |
| `/auth/refresh` | POST | Refresh Token | Get new access token |
| `/auth/logout` | POST | Access Token | Invalidate refresh token |
| `/auth/me` | GET | Access Token | Get current user |
| `/auth/password` | PUT | Access Token | Change password |

---

## 3. Response Format

### 3.1 Success Response
```json
{
  "data": { ... },
  "meta": {
    "page": 1,
    "per_page": 25,
    "total": 100,
    "total_pages": 4
  }
}
```

### 3.2 Error Response
```json
{
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "Invalid request data",
    "details": [
      {
        "field": "email",
        "message": "Invalid email format"
      }
    ]
  }
}
```

### 3.3 HTTP Status Codes

| Code | Meaning | Usage |
|------|---------|-------|
| 200 | OK | Successful GET, PUT |
| 201 | Created | Successful POST |
| 204 | No Content | Successful DELETE |
| 400 | Bad Request | Validation error |
| 401 | Unauthorized | Missing/invalid token |
| 403 | Forbidden | Insufficient permissions |
| 404 | Not Found | Resource doesn't exist |
| 409 | Conflict | Duplicate resource |
| 422 | Unprocessable | Business logic error |
| 500 | Server Error | Internal error |

## 4. Inventory Endpoints

### 4.1 List Inventory
```
GET /inventory
```

**Query Parameters:**
| Param | Type | Default | Description |
|-------|------|---------|-------------|
| q | string | - | Search query |
| category | string | - | Filter by category |
| status | enum | - | in_stock, low_stock, out_of_stock |
| warehouse_id | uuid | - | Filter by warehouse |
| page | int | 1 | Page number |
| per_page | int | 25 | Items per page |
| sort_by | string | title | Sort field |
| sort_order | string | asc | asc or desc |

**Response (200):**
```json
{
  "data": [
    {
      "id": "uuid",
      "isbn": "978-0-13-235088-4",
      "title": "Clean Code",
      "author": "Robert C. Martin",
      "category": "Programming",
      "mfr_price": "35.00",
      "list_price": "49.99",
      "total_quantity": 45,
      "incoming_quantity": 20,
      "status": "in_stock"
    }
  ],
  "meta": { "page": 1, "total": 100 }
}
```

**Auth:** Admin, Employee  
**Task Ref:** S2-7, S2-11

## 5. Sales Endpoints

### 5.1 Create Sale
```
POST /sales
```

**Request Body:**
```json
{
  "customer_id": "uuid (optional)",
  "line_items": [
    {
      "book_id": "uuid",
      "quantity": 2,
      "unit_price": "49.99"
    }
  ],
  "payment_method": "card",
  "discount_amount": "0.00"
}
```

**Response (201):**
```json
{
  "data": {
    "id": "uuid",
    "transaction_number": "TXN-00001234",
    "subtotal": "99.98",
    "tax_amount": "8.25",
    "total": "108.23",
    "created_at": "2026-05-05T14:30:00Z"
  }
}
```

**Auth:** Admin, Employee  
**Story Ref:** US-006

---

### 5.2 List Sales
```
GET /sales
```

**Query Parameters:**
| Param | Type | Description |
|-------|------|-------------|
| start_date | datetime | Filter start |
| end_date | datetime | Filter end |
| customer_id | uuid | Filter by customer |
| employee_id | uuid | Filter by employee |
| book_id | uuid | Filter by book |
| payment_method | enum | cash, card, account |

**Auth:** Admin, Employee

---

### 5.3 Sales Reports
```
GET /sales/reports/daily?date=2026-05-05

GET /sales/reports/monthly?month=2026-05
GET /sales/reports/bestsellers?start_date=...&end_date=...
```

**Auth:** Admin, Employee

## 10. Pagination

All list endpoints support pagination:

```
GET /inventory?page=2&per_page=50
```

**Response Meta:**
```json
{
  "meta": {
    "page": 2,
    "per_page": 50,
    "total": 1234,
    "total_pages": 25,
    "has_next": true,
    "has_prev": true
  }
}
```
